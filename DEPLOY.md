# Cavree Deployment Guide (AWS Amplify + RDS + Cloudflare)

This guide deploys Cavree using AWS Amplify Hosting for the Next.js app, AWS RDS PostgreSQL for the database, and Cloudflare for DNS.

---

## Production URL Map

| Purpose | URL |
|---|---|
| Franchise Application | `https://cavree.com/franchise/apply` |
| Franchise Panel | `https://franchise.cavree.com/admin/dashboard` |
| Admin Panel | `https://admin.cavree.com/admin/dashboard` |
| Super Admin Panel | `https://super-admin.cavree.com/super-admin/dashboard` |
| Single Login Page | `https://cavree.com/auth/login` |

## Prerequisites

1. AWS account access.
2. Cloudflare account managing DNS for `cavree.com`.
3. GitHub repository access: `https://github.com/cavreeaws-lab/cavree`.
4. Production database password.
5. `NEXTAUTH_SECRET` value.
6. Razorpay keys. Dummy values can be used first and replaced later.

---

## Step 1: Create AWS RDS PostgreSQL

Go to AWS Console → RDS → Databases → Create database.

Recommended settings:

| Setting | Value |
|---|---|
| Engine | PostgreSQL |
| Template | Production or Dev/Test |
| DB identifier | `cavree-prod-db` |
| Master username | `cavree_admin` |
| Master password | Use a strong password |
| Instance class | `db.t4g.micro` or larger |
| Storage | 20 GB gp3 |
| Storage autoscaling | Enabled |
| Initial database name | `cavree` |
| Region | `ap-south-1` |

For the first deployment, enable public access if Amplify must connect directly to RDS. Restrict access later when production traffic is verified.

### 1.1 RDS Security Group

Open the RDS security group and add an inbound rule:

| Type | Port | Source |
|---|---:|---|
| PostgreSQL | `5432` | Your trusted IP for local migrations |

If Amplify build needs to run Prisma migrations directly and cannot reach RDS, temporarily allow PostgreSQL access from `0.0.0.0/0`, deploy, then restrict it immediately. A safer long-term option is running migrations from a trusted machine or a controlled AWS environment.

### 1.2 Database URL

After RDS is available, copy the endpoint and create:

```env
DATABASE_URL="postgresql://cavree_admin:YOUR_DB_PASSWORD@YOUR_RDS_ENDPOINT:5432/cavree?schema=public"
```

## Step 2: Create AWS Amplify App

Go to AWS Console → Amplify → Create new app → Deploy from Git repository.

Use:

| Setting | Value |
|---|---|
| Provider | GitHub |
| Repository | `cavreeaws-lab/cavree` |
| Branch | `main` |
| Framework | Next.js |
| Region | `ap-south-1` |

Authorize GitHub access when prompted.

---

## Step 3: Amplify Build Settings

Use these build settings in Amplify:

```yaml
version: 1
frontend:
  phases:
    preBuild:
      commands:
        - npm ci
        - npx prisma generate
    build:
      commands:
        - npx prisma migrate deploy
        - npm run build
  artifacts:
    baseDirectory: .next
    files:
      - '**/*'
  cache:
    paths:
      - node_modules/**/*
      - .next/cache/**/*
```

If `npx prisma migrate deploy` fails because Amplify cannot connect to RDS, remove it from the build step and run migrations from a trusted local machine:

```bash
npx prisma migrate deploy
```

## Step 4: Amplify Environment Variables

In Amplify → App settings → Environment variables, add:

```env
DATABASE_URL=postgresql://cavree_admin:YOUR_DB_PASSWORD@YOUR_RDS_ENDPOINT:5432/cavree?schema=public
NEXTAUTH_URL=https://cavree.com
NEXTAUTH_SECRET=GENERATE_A_STRONG_SECRET
MAIN_DOMAIN=cavree.com
NEXT_PUBLIC_APP_URL=https://cavree.com

RAZORPAY_KEY_ID=rzp_test_dummy_key
RAZORPAY_KEY_SECRET=rzp_test_dummy_secret

AWS_REGION=ap-south-1
AWS_ACCESS_KEY_ID=dummy
AWS_SECRET_ACCESS_KEY=dummy

S3_BUCKET_NAME=cavree-images
CLOUDFRONT_URL=https://cdn.cavree.com

FROM_EMAIL=noreply@cavree.com
ADMIN_EMAIL=admin@cavree.com
NODE_VERSION=20
```

Generate `NEXTAUTH_SECRET` locally:

```bash
openssl rand -base64 32
```

Replace dummy Razorpay keys with live values before enabling real payments.

## Step 5: Add Custom Domains in Amplify

Go to Amplify → Hosting → Custom domains → Add domain.

Add:

| Domain | Branch |
|---|---|
| `cavree.com` | `main` |
| `www.cavree.com` | `main` |
| `admin.cavree.com` | `main` |
| `franchise.cavree.com` | `main` |
| `super-admin.cavree.com` | `main` |

Amplify will generate DNS validation and routing records. Copy those values into Cloudflare.

---

## Step 6: Configure Cloudflare DNS

Add the DNS records generated by Amplify.

Typical records:

| Type | Name | Target | Proxy Status |
|---|---|---|---|
| CNAME or flattened CNAME | `@` | Amplify target | DNS only during verification |
| CNAME | `www` | Amplify target | DNS only during verification |
| CNAME | `admin` | Amplify target | DNS only during verification |
| CNAME | `franchise` | Amplify target | DNS only during verification |
| CNAME | `super-admin` | Amplify target | DNS only during verification |
| CNAME | AWS validation name | AWS validation target | DNS only |

Cloudflare SSL/TLS settings:

- Mode: **Full**
- Always Use HTTPS: **ON**
- Proxy status: keep **DNS only** until Amplify verifies the domain and SSL certificate.

After verification, you can test enabling the Cloudflare proxy. If Amplify domain validation or SSL renewal has issues, switch the records back to DNS only.

## Important Production Note: Single Login Across Subdomains

The app uses `cavree.com/auth/login` as the single login page. For that login session to work on all subdomains, the session cookie must be available to `.cavree.com`.

If users can log in on `cavree.com` but are still redirected to login on `admin.cavree.com`, `franchise.cavree.com`, or `super-admin.cavree.com`, update the session cookie configuration to set the production cookie domain as `.cavree.com`.

## Troubleshooting

### Amplify build fails during Prisma migration

- Confirm RDS is available.
- Confirm `DATABASE_URL` is correct.
- Confirm RDS security group allows the migration runner.
- Remove migration from Amplify build and run `npx prisma migrate deploy` locally.

### Subdomains do not open

- Confirm Amplify custom domain setup is complete.
- Confirm Cloudflare CNAME records match Amplify.
- Keep Cloudflare proxy disabled until Amplify SSL is verified.

### Login works on main domain but not subdomains

- Confirm the production session cookie domain is `.cavree.com`.
- Confirm `MAIN_DOMAIN=cavree.com`.
- Confirm `NEXTAUTH_URL=https://cavree.com`.

### Payment does not work

- Replace dummy Razorpay keys with live keys.
- Redeploy the Amplify app after changing environment variables.
